## القسم 3 — التحسينات المقترحة

### تحسينات عامة (مرتبة بالأولوية)
1. **أتمتة البناء والاختبار** — لا يوجد أي اختبار في المشروع ولا CI. أضف GitHub Actions (build + lint) واختبارات وحدة لـ `PrayerTimeCalculator` و`DatabaseSeeder` والتاريخ الهجري.
2. **إشعارات الأذكار** — لا يوجد حالياً إلا إشعار الأذان. أضف تذكير أذكار الصباح (بعد الفجر) والمساء (بعد العصر) والنوم (وقت يحدده المستخدم) — هذا جوهر تطبيق أذكار (مفصل في القسم 4).
3. **ربط `OfflineBanner` و`RatingDialog`** — موجودان جاهزان وغير مستخدمين؛ الأول مهم لشاشة المشغل الصوتي، والثاني لنمو التقييمات في المتجر.
4. **Widget الشاشة الرئيسية** — `PrayerWidget` موجود؛ أضف widget ذكر اليوم/عداد التسبيح، فهو أكبر قناة عودة يومية.
5. **نسخ احتياطي حقيقي** — `allowBackup=true` غير كافٍ؛ أكمل التصدير/الاستيراد JSON (يحل المشكلة الحرجة #2 أيضاً).

### تحسينات UI محددة (لجذب المستخدم عالمياً)
1. **خلفية سماء حية حسب الوقت** — `PrayerSkyBackground` موجود بالفعل وغير مستخدم! اربطه ببطاقة المواقيت بحيث يتدرج اللون (فجر بنفسجي → ظهر أزرق → مغرب برتقالي → ليل داكن). أثر بصري كبير بتكلفة صفر تقريباً.
2. **انتقال احتفالي موحد** — عند إتمام الأذكار، استخدم `CelebrationScreen` مع اهتزاز خفيف (haptics) وصوت اختياري — لحظة "المكافأة" هي أهم لحظة في تطبيق العادات.
3. **وضع قراءة ليلي دافئ للقرآن** — خلفية داكنة مائلة للدفء (وليست سوداء) مع خيار حجم خط خاص بالمصحف.
4. **صفحة المصحف بدل القائمة الطويلة** — عرض صفحات حقيقية (604 صفحات) مع تقليب أفقي، فالحفّاظ يتعاملون بالصفحة لا بالتمرير.
5. **Onboarding قصصي من 3 شاشات** — (مرحباً → حدد موقعك لمواقيت دقيقة → فعّل التذكيرات) مع طلب الصلاحيات في سياقها.
6. **دعم الأجهزة اللوحية فعلياً** — شاشات tablet موجودة يتيمة؛ اربطها بـ `WindowSizeClass` أو احذفها.

**القاعدة الذهبية التي تربط الشرع بعلم العادات:** قوله ﷺ: **«أحبُّ الأعمال إلى الله أدومُها وإن قلّ»** (متفق عليه) — وهذا حرفياً مبدأ "العادات الصغيرة المستدامة" في علم السلوك الحديث.

### 4.2 الأساس العلمي (علم بناء العادات)

1. **الربط بالمراسي (Habit Stacking / Anchoring)** — بحوث BJ Fogg في ستانفورد (Tiny Habits): أنجح طريقة لبناء عادة هي ربطها بسلوك يومي ثابت. **المسلم يملك 5 مراسٍ يومية مثالية: الصلوات الخمس** — وهي مراسٍ متحركة مع الشمس، والتطبيق يحسبها أصلاً. الصيغة: "بعد صلاة الفجر → أذكار الصباح".
2. **نوايا التنفيذ (Implementation Intentions)** — التحليل الشمولي لـ Gollwitzer & Sheeran (2006، 94 دراسة، +8000 مشارك): صياغة "إذا حدث X أفعل Y" تضاعف الالتزام 2–3 مرات مقارنة بالنية العامة. التطبيق يجعل المستخدم يختار خطته: "متى ستقرأ أذكار النوم؟" بدلاً من تذكير عام.
3. **البدايات الصغيرة (Tiny Habits)** — البدء بحد أدنى لا يفشل (3 أذكار أساسية) ثم التوسع، بدل قائمة كاملة تسبب الإحباط والهجر — وهو عين حديث «أدومها وإن قل» و«اكلَفوا من العمل ما تُطيقون» (متفق عليه).
4. **السلسلة مع الرحمة** — سلاسل الأيام (streaks) محفزة (النفور من الخسارة)، لكن البحوث تحذر: كسر السلسلة بدون "رحمة" يؤدي للهجر الكامل. الحل: "رخصة يوم" شهرية تحفظ السلسلة، ورسالة لطيفة لا لوم فيها («يا أيها الناس توبوا إلى الله» — الباب مفتوح دائماً).
5. **المكافأة الفورية والاحتفال** — لحظة الإتمام تحتاج احتفالاً حسياً فورياً (موجود: CelebrationScreen + الحديقة الروحية) مع عرض **الفضل الوارد** للذكر نفسه ("حُطَّت خطاياه وإن كانت مثل زبد البحر") — فالمكافأة الحقيقية معنوية.

### 4.3 تصميم الميزة المقترح

**«رفيق اليوم» — خط زمني حي يرافق المستخدم من الفجر إلى النوم:**

1. **بطاقة "الآن" الذكية** (أعلى الرئيسية): تعرف الوقت الحالي نسبةً للمواقيت المحسوبة وتقترح العمل المناسب لحظياً — بعد الفجر: أذكار الصباح؛ ضحىً: ركعتا الضحى؛ بعد أي صلاة: أذكار ما بعد الصلاة؛ بعد العصر: أذكار المساء؛ الجمعة: الكهف؛ قبل النوم: أذكار النوم.
2. **شاشة الخط الزمني**: 8 محطات لليوم (استيقاظ → فجر+صباح → ضحى → ظهر → عصر+مساء → مغرب → عشاء → نوم)، كل محطة بطاقة فيها: الذكر بنصه الكامل، المصدر والدرجة، **الفضل**, عداد باللمس، وحالة إتمام. المحطة النشطة تتوهج، والمنتهية تظهر بعلامة.
3. **إشعارات مربوطة بالمواقيت الفعلية** (وليس ساعات ثابتة): بعد الفجر بـ 10 دقائق، بعد العصر بـ 15 دقيقة، وقت نوم يحدده المستخدم — مع إعادة جدولة يومية تلقائية (يحل المشكلة الحرجة #7 معاً).
4. **وضع المداومة (تطبيق «أدومها وإن قل»)**: المستخدم يختار "الحد الأدنى اليومي" (مثلاً 3 أذكار صباح + 3 مساء) — إتمامه يحفظ السلسلة، وكل زيادة تظهر كـ"فضل". لا لوم عند التقصير أبداً؛ رخصة شهرية تحفظ السلسلة.
5. **الربط بالموجود**: كل محطة تُسجل في `DailyProgress` (توسيع الجدول بأعمدة: `dhuha_done`, `post_prayer_adhkar`, `sleep_adhkar_done`, `istighfar_count`) وتغذي الحديقة الروحية والإنجازات والتقرير الأسبوعي الموجودين — بدون بناء أنظمة موازية.
6. **محتوى الجمعة الخاص**: تنبيه الكهف + عداد صلاة على النبي ﷺ + تذكير ساعة الإجابة قبيل المغرب.
